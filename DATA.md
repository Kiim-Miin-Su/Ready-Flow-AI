## 2. 핵심 설계 결정 (사용자 원안 대비 수정점)

원안은 타당하나, **분류 학습이 성립하려면** 아래를 보완했습니다. 근거와 함께 명시합니다.

### (A) 분석 단위 = (법정동 × 날짜) 패널 — *가장 중요한 수정*
침수흔적도는 **양성(=침수)만 있는 데이터**라 그대로는 분류가 불가능합니다(음성 부재).
→ **(동, 날짜)** 격자를 만들고 그 날 침수가 있으면 `y=1`, 없으면 `y=0`. 이것이 "언제·어디·얼마나 비올 때 침수"라는 질문 그 자체입니다.

- **동 모집단** = 흔적도에 등장한 침수 이력 동들의 합집합 (2023∪2024) = **93개 법정동**. (침수 위험군. 한 번도 안 잠긴 동까지 넣으려면 외부 법정동 마스터가 필요하고 클래스가 극단적으로 불균형해짐 → §7.)
- **날짜 모집단** = 해당 동이 속한 구의 **일강우 ≥ 1.0mm** 인 날 ∪ 실제 침수일. (마른 날 음성 폭증 방지 + 양성 누락 방지.)
- 결과 패널: **14,690 행 · 양성 164 (1.12%)**.

### (B) 강우 → 동 조인: "동 단위 불가능" 문제의 실제 해법
관측소는 48곳뿐인데 서울 법정동은 ~424개 → **물리적으로 대부분 동에 자체 관측소가 없음**. 따라서 *진짜* 동 해상도 강우는 불가능. 대신:

- **모든 25개 구에 `○○구청` 관측소가 1개씩 존재함**(검증 완료, NFC 정규화 후). → 각 동을 **자기 구의 구청 관측소**에 조인. 1구=1대표계열로 결측 없는 깔끔한 구 해상도.
- 나머지 23개 펌프장/동 관측소(예: `면목펌프장`→중랑구, `목동펌프장`→양천구)는 사용자 요청대로 **위치를 동/구로 식별**해 `STATION_GU` 매핑표에 기록함(코드 내). v1에서는 구청 계열을 대표로 쓰고, 이들은 **최근접 관측소 업그레이드용**으로 남겨둠(§7).
- **동 정체성은 보존**: `ADM_CD` one-hot/노드 ID로 동 단위 식별은 그대로 유지. (강우 해상도 = 구, 동 식별 = 동.)

> macOS 파일명은 **NFD(분해형)**라 `'구청' in name` 이 그냥은 실패함 → 전 구간 `unicodedata.normalize('NFC', …)` 적용.

### (C) 날짜 신뢰성
`F_YR`/폴더연도 ≠ 사건일. (2024 폴더에 `F_SAT_YMD=2023…` 인 보험 레코드 존재.)
→ **`F_SAT_YMD`를 사건일로 사용**, 연·구 범위(2023–2024, 서울)로 필터. 시간 컬럼은 `format='mixed'`로 파싱(일부 행 초 단위 포함).

### (D) 누수(leakage) 차단
모든 이력 피처는 **사건일 t 직전까지의 데이터만** 사용(`searchsorted`로 strict-prior). 동시점 침수율·이웃율을 그대로 쓰면 라벨 누설 → 금지.

### (E) 서빙 피처 패리티 (★ 실시간 서빙 전제의 핵심 제약)
서빙 입력은 **주소+예보**뿐. 추론 시점에 **얻을 수 없는 피처는 학습에 넣으면 안 됨**.
→ 피처를 "serve-safe"와 "학습 전용/주의"로 §5에서 명확히 구분.

## 6. 건물 유형 & 지형 — 주의/외부데이터

**건물 유형(`TYPE`)**: 사용자 지적대로 중요하나, 흔적도의 `TYPE`은 *침수된 건물*만 기록 → (a) 음성 동엔 값이 없고 (b) 라벨 파생이라 그대로 쓰면 **누수**. 또 서빙 시 주소만으론 알 수 없음.
→ v1에서는 동별 **건물구성 비율**(`btype_frac_residential/commercial/…`)을 *약한 정적 prior*로 `dong_static`에만 보관(모델 행렬 기본 제외). 제대로 쓰려면 **외부 건물DB**(국가공간정보 GIS건물통합정보: 동별 용도·반지하 비율 등) 조인 권장. 서빙에선 "동 우세 유형" 또는 사용자 입력으로 제공.

**지형(하천거리·평균고도·배수밀도)**: `./data`에 없음. `F_AVR_HGT`는 침수지점에만 있어 누수·음성결측 → **미사용**. `dong_static`에 NaN 컬럼으로 자리만 마련.
권장 외부 소스(동 단위 1회 구축, 정적이라 서빙 친화적):
- **평균고도/표고기복**: 국토지리정보원 **DEM**(5m) → 동 경계 내 평균.
- **하천거리**: 하천망 shp → 동 중심/경계까지 최단거리.
- **배수밀도**: 우수관거/빗물펌프장 GIS → 동 면적당 시설 수·관거 연장.

## 8. 모델 권고 — GNN vs scikit-learn (Vercel 무료/CPU)

**결론: v1은 scikit-learn 권장. GNN은 비권장(특히 Vercel 무료).**

| 항목 | 현재 데이터 | 함의 |
|---|---|---|
| 노드(동) | **93개** | 그래프가 너무 작음 |
| 엣지 | 동일구 클리크 **프록시** | 진짜 인접 아님 → GNN 메시지패싱 이득 미미 |
| 양성 | **164** | 딥모델 과적합 위험 |
| 피처 | ~20개 정형 | **트리/부스팅에 최적** |

**Vercel 무료의 결정적 제약 — 컴퓨트가 아니라 번들 크기:**
- `torch + torch-geometric` ≈ **800MB+** → Vercel 서버리스 함수 **250MB(unzip) 한도 초과** → 배포 불가.
- 단, `scikit-learn`(scipy ~130MB 포함)도 250MB 초과 → **학습모델을 평문 JSON으로 내보내 numpy로만 서빙**(~65MB). 상세 `INFRA.md §2`, `MODEL.md §2`.

**권장 스택**
- 모델: **`HistGradientBoostingClassifier`**(또는 LightGBM/XGBoost). 불균형은 `class_weight='balanced'`/`scale_pos_weight`.
- 검증: **시간 분할**(2023 train → 2024 test)로 누수 없는 일반화 측정. PR-AUC·Recall 중심.
- 서빙(FastAPI): 학습모델을 평문 JSON(`model_np.json`)으로 내보내 **numpy로만 평가**(sklearn/scipy 불필요). 기동 시 모델 + 동별 정적 테이블 로드 → 요청(주소+예보 강우)으로 `rain_*` 계산 → 확률. CPU 단건 추론 수 ms. (배포 상세 `INFRA.md`, 모델 `MODEL.md`)

> GNN 재고 시점: 진짜 법정동 인접 그래프 + 동 정적 피처(지형) + 더 많은 연도가 쌓였을 때. 그때도 **Vercel 무료가 아닌 컨테이너 호스트**(torch 용량)에서 서빙.
